# Use readable Markdown filenames as note identity

Calmd will represent notes with readable Markdown filenames such as `purification.md`, rather than opaque IDs. Renaming the filename is a rename of the same note, preserving its content and conceptual identity; internal links to it should update automatically. This keeps the filesystem understandable outside Calmd, while requiring deliberate handling of filename collisions and links when filesystem integration is introduced.

This follows Obsidian’s file-first model: a vault is a local folder of Markdown files that can be edited with external tools, and renaming a note updates links to it. The Markdown files are the source of truth; search indexes, metadata caches, and semantic indexes are derived and must be rebuildable. Calmd will reject note titles that cannot safely be represented as filenames rather than silently sanitizing them. Validation will use a conservative cross-platform rule set so a vault remains portable between Linux, macOS, and Windows. References: [Vault settings](https://obsidian.md/help/data-storage), [Create a new note](https://obsidian.md/help/create-note), and [Supported formats for internal links](https://obsidian.md/help/links).
